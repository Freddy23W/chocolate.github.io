# Documentation

This repository is intended to be used as a **template**.

## Template layer vs project layer

- **Template/workflow layer**: `orga/`, `.windsurf/`, `docs/windsurf/`, and GitHub Actions automation.
- **Project layer**: your application code (typically `src/`, `app.py`, `tests/`, `templates/`) and project-specific docs.

## Start here

- `docs/initializing-a-project.md` – how to initialize a new project from this template.
- `docs/task-workflow.md` – how tasks map to branches/PRs and orga task folders.
- `docs/github-actions-setup.md` – required GitHub settings so Actions can create branches and PRs.
- `docs/windsurf/` – Windsurf/Cascade copy-pack (rules, workflows, skills) and sync policy.

## Examples

- `docs/example_program.json` – machine-parseable example data file.

## Notes

Some files in `docs/` may still reflect a previously-copied project and are candidates for cleanup or removal as part of turning this into a fully generic template.
