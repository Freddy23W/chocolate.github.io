# Project Orga

This folder collects all **non-code project organization** for this repository.

## Start here

- `tasks/TASKS.md` – overview of active/backlog/archived tasks.
- `tasks/active/` – current task folders (one folder per task).
- `tasks/archive/` – completed tasks and their notes.
- `domain/` – domain and architecture notes (requirements, data model, architecture, project plan).
- `../docs/windsurf/` – template/reference for AI/Windsurf rules and workflows (active ones live under `.windsurf/`).

Note: `orga/_legacy/` contains reference/example content from earlier work. New projects should track current work under `orga/tasks/active/`.

## Conventions

- Each task has an ID like `T-001` and its own folder under `tasks/active/`.
- When done, a task folder is moved to `tasks/archive/` and `TASKS.md` is updated.
