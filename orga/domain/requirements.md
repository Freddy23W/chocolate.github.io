# Requirements

## Functional requirements
- **FR-1**: Landing page loads when QR code is scanned ✅
- **FR-2**: Audio file plays automatically on page load ✅
- **FR-3**: Audio controls are visible and functional ✅
- **FR-4**: Page displays festival branding and information ✅
- **FR-5**: Responsive design works on mobile devices ✅
- **FR-6**: Dropdown navigation to select between 4 chapters
- **FR-7**: Text content display for each chapter
- **FR-8**: Navigation back to landing page from any chapter
- **FR-9**: Chapter-specific audio files with autoplay
- **FR-10**: Mobile-optimized chapter selection interface

## Non-functional requirements
- **NFR-1**: Page loads in under 3 seconds ✅
- **NFR-2**: Audio playback works on major browsers (Chrome, Safari, Firefox) ✅
- **NFR-3**: Mobile-first responsive design ✅
- **NFR-4**: Graceful handling of browser autoplay restrictions ✅
- **NFR-5**: No user data collection or tracking ✅
- **NFR-6**: Smooth transitions between chapters
- **NFR-7**: Touch-friendly navigation on mobile devices
- **NFR-8**: Consistent performance across all chapters

## Acceptance criteria
- AC1: Scanning QR code opens a beautiful landing page ✅
- AC2: Audio plays automatically (or with single tap) ✅
- AC3: Page works on iOS and Android devices ✅
- AC4: Audio can be controlled by the user ✅
- AC5: Design reflects queer festival aesthetics ✅
- AC6: Dropdown allows selection of 4 chapters
- AC7: Each chapter displays unique text content
- AC8: Each chapter has associated audio that plays automatically
- AC9: Users can easily return to landing page from any chapter
- AC10: Navigation is intuitive and mobile-friendly
